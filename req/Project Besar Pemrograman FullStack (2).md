Ada project ini, mahasiswa diminta untuk membangun sebuah aplikasi web fullstack berbasis Node.js yang dapat menyelesaikan suatu permasalahan nyata. Mahasiswa bebas memilih topik aplikasi (misalnya sistem penjualan, booking, antrian, manajemen tugas, dll), namun wajib mengikuti struktur yang telah ditentukan.

Tujuan dari project ini adalah agar mahasiswa mampu memahami proses pengembangan aplikasi secara menyeluruh, mulai dari perancangan sistem, implementasi, hingga deployment.

**1. Latar Belakang & Tujuan Sistem**

Mahasiswa harus menjelaskan:

- Masalah apa yang ingin diselesaikan
- Mengapa sistem ini penting
- Siapa pengguna sistem

**2. Arsitektur Aplikasi (WAJIB)**

Mahasiswa harus menjelaskan:

- Struktur aplikasi (Frontend, Backend, Database)
- Alur sistem (request → server → database → response)
- Diagram arsitektur sederhana

Contoh:

- Frontend (React / HTML / Vue, Bootstrap)
- Backend (Node.js / Express)
- Database (MySQL)

**3. Desain Database (ERD / Schema)**

Mahasiswa wajib:

- Membuat ERD (untuk SQL) atau schema design (NoSQL)
- Menjelaskan relasi data

Ketentuan:

- Minimal 3 entitas / collection
- Harus ada relasi/logika antar data

**4. Implementasi Aplikasi (Fullstack)**

**Backend:**

- Menggunakan Node.js (disarankan Express.js)
- Membuat API (REST API)
- Mengelola routing dan logic

**Frontend:**

- Bebas (React / Vue / HTML / template)
- Tampilan harus user-friendly



**5. Authentication & Authorization**

Aplikasi harus memiliki:

- Login & logout
- Minimal 2 role (Admin & User)
- Hak akses berbeda

Disarankan:

- JWT (JSON Web Token)

**6. CRUD & Business Logic**

Mahasiswa harus membuat:

- CRUD (Create, Read, Update, Delete)
- Logic tambahan

Contoh:

- Perhitungan otomatis
- Status dinamis
- Validasi data

**7. Dashboard**

Aplikasi harus memiliki:

- Ringkasan data
- Informasi utama sistem
- (Opsional) grafik

**8. API & Integrasi**

Mahasiswa wajib:

- Membuat REST API
- Menggunakan endpoint yang jelas

Opsional:

- Integrasi API eksternal

**9. DevOps (WAJIB - DASAR)**

**a. Version Control**

- Menggunakan Git & GitHub
- Memiliki commit history yang jelas

Seluruh hasil project yang telah dikerjakan wajib disusun dalam bentuk **laporan (format PDF)**. Laporan ini berfungsi untuk menjelaskan proses pengembangan aplikasi secara lengkap, mulai dari perencanaan hingga deployment.
